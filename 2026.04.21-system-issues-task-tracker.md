## Task 01: Inability to Edit Member Details in Group View (Role Validation Error)

- Status: Closed

### Description
When a VSLA Facilitator opens My Groups and edits an existing group member, updates fail due to a validation error requiring the role field.

### Steps to Reproduce
1. Log in as a VSLA Facilitator.
2. Navigate to VSLA Facilitator > My Groups.
3. Open any group.
4. Attempt to edit an existing member record.

### Observed Behaviour
A validation error appears: "The role field is required," and the member update is rejected.

### Expected Behaviour
Users should be able to update member details without role-related validation failures unless role is intentionally mandatory and correctly presented in the form.

### Developer Remarks
- Cause was confirmed and fixed.
- Member editing now works without the false role error.
- Final checks confirmed the update flow is working correctly.
